# 1.1 用户管理设计文档

> 版本：v1.0 | 日期：2026-04-14 | 所属阶段：阶段1 - 系统基础

## 1. 功能概述

**目标：** 建立基于数据库的用户体系，替换当前 AuthService 中的硬编码 admin/123456 认证，支持管理员对用户进行增删改查，并提供密码管理和个人信息维护能力。

**后续依赖此模块：** 角色权限（1.2）、组织架构（1.3）、菜单动态权限（1.4）、登录认证（1.5）、审批流审批人规则。

## 2. 数据库设计

### 2.1 nscp_sys_user（用户表）

```sql
CREATE TABLE nscp_sys_user (
    id                  BIGINT          NOT NULL AUTO_INCREMENT PRIMARY KEY COMMENT '主键',
    tenant_id           VARCHAR(64)     NOT NULL                           COMMENT '租户ID',
    username            VARCHAR(64)     NOT NULL                           COMMENT '登录用户名（租户内唯一）',
    password_hash       VARCHAR(256)    NOT NULL                           COMMENT '密码（BCrypt哈希，严禁明文）',
    real_name           VARCHAR(64)     NOT NULL                           COMMENT '真实姓名',
    email               VARCHAR(128)                                       COMMENT '邮箱',
    phone               VARCHAR(32)                                        COMMENT '手机号',
    avatar_url          VARCHAR(512)                                       COMMENT '头像URL',
    user_type           VARCHAR(32)     NOT NULL DEFAULT 'NORMAL'          COMMENT '用户类型：SYSTEM_ADMIN/TENANT_ADMIN/NORMAL',
    status              VARCHAR(32)     NOT NULL DEFAULT 'ACTIVE'          COMMENT '状态：ACTIVE/DISABLED/LOCKED',
    locked_until        DATETIME                                           COMMENT '锁定解锁时间点（NULL 表示未锁定）',
    pwd_changed_at      DATETIME                                           COMMENT '密码最后修改时间',
    created_by          VARCHAR(64)     NOT NULL                           COMMENT '创建人ID',
    created_at          DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_by          VARCHAR(64)                                        COMMENT '最后修改人ID',
    updated_at          DATETIME                 ON UPDATE CURRENT_TIMESTAMP COMMENT '最后修改时间',
    deleted             TINYINT(1)      NOT NULL DEFAULT 0                 COMMENT '逻辑删除（0=未删/1=已删）',

    UNIQUE KEY uk_tenant_username (tenant_id, username),
    INDEX idx_tenant_status (tenant_id, status),
    INDEX idx_tenant_deleted (tenant_id, deleted)
) COMMENT '系统用户表';
```

**字段说明：**

| 字段 | 说明 |
|------|------|
| user_type | SYSTEM_ADMIN：超级管理员（跨租户）；TENANT_ADMIN：租户管理员；NORMAL：普通用户 |
| status | ACTIVE：正常；DISABLED：管理员手动禁用；LOCKED：密码连续错误触发锁定 |
| locked_until | 锁定结束时间，`NULL` 或过去时间表示未锁定。到期自动解锁（登录时判断） |
| password_hash | BCrypt 哈希（bcrypt strength=12）。**严禁存明文** |
| pwd_changed_at | 用于密码过期检查；初始设为创建时间 |

## 3. 后端接口设计

### 3.1 接口列表

| 方法 | 路径 | 所需权限 | 功能 |
|------|------|---------|------|
| GET | `/api/sys/users` | `sys:user:list` | 分页查询用户列表 |
| POST | `/api/sys/users` | `sys:user:create` | 新增用户 |
| GET | `/api/sys/users/{id}` | `sys:user:read` | 查询单个用户详情 |
| PUT | `/api/sys/users/{id}` | `sys:user:update` | 修改用户基本信息 |
| DELETE | `/api/sys/users/{id}` | `sys:user:delete` | 删除用户（逻辑删除） |
| POST | `/api/sys/users/{id}/enable` | `sys:user:update` | 启用用户 |
| POST | `/api/sys/users/{id}/disable` | `sys:user:update` | 禁用用户 |
| POST | `/api/sys/users/{id}/reset-password` | `sys:user:reset-pwd` | 重置为默认密码 |
| GET | `/api/sys/users/me` | 已登录 | 获取当前用户信息 |
| PUT | `/api/sys/users/me/profile` | 已登录 | 修改个人信息（姓名/邮箱/手机） |
| PUT | `/api/sys/users/me/password` | 已登录 | 修改自己密码 |

### 3.2 请求/响应格式

#### 分页查询用户列表

**请求：** `GET /api/sys/users?page=1&size=20&username=zhang&status=ACTIVE`

| 参数 | 类型 | 说明 |
|------|------|------|
| page | int | 页码，从 1 开始，默认 1 |
| size | int | 每页条数，默认 20，最大 100 |
| username | string | 用户名模糊搜索（可选） |
| realName | string | 姓名模糊搜索（可选） |
| status | string | 状态筛选（可选） |

**响应：**
```json
{
  "code": 0,
  "data": {
    "total": 100,
    "page": 1,
    "size": 20,
    "records": [
      {
        "id": 1,
        "username": "zhangsan",
        "realName": "张三",
        "email": "zhangsan@example.com",
        "phone": "138****0001",
        "userType": "NORMAL",
        "status": "ACTIVE",
        "deptId": 10,
        "deptName": "研发部",
        "roles": [
          { "id": 1, "roleName": "供应链分析师" }
        ],
        "createdAt": "2026-04-14T10:00:00"
      }
    ]
  },
  "msg": "success"
}
```

#### 新增用户

**请求：** `POST /api/sys/users`
```json
{
  "username": "zhangsan",
  "realName": "张三",
  "email": "zhangsan@example.com",
  "phone": "13800000001",
  "userType": "NORMAL",
  "deptId": 10,
  "roleIds": [1, 2]
}
```

**业务规则：**
1. 校验 username 在当前 tenant_id 内唯一
2. 初始密码设为系统配置的默认值，BCrypt 哈希后存储
3. 同步写入 nscp_sys_user_role（用户-角色关联）
4. 同步写入 nscp_sys_user_dept（用户-部门关联）
5. pwd_changed_at 设为当前时间（触发首次登录强制改密，可选策略）

#### 修改用户

**请求：** `PUT /api/sys/users/{id}`
```json
{
  "realName": "张三改",
  "email": "new@example.com",
  "phone": "13900000002",
  "userType": "NORMAL",
  "deptId": 11,
  "roleIds": [2, 3]
}
```

**业务规则：**
- username 不允许修改
- roleIds 更新时先删旧关联再插新关联（事务）

#### 重置密码

**请求：** `POST /api/sys/users/{id}/reset-password`（无请求体）

**响应：** 返回新的初始密码明文（仅此一次展示，管理员告知用户）

#### 修改自己密码

**请求：** `PUT /api/sys/users/me/password`
```json
{
  "oldPassword": "Admin@2026",
  "newPassword": "MyNew@2026",
  "confirmPassword": "MyNew@2026"
}
```

**业务规则：**
1. oldPassword BCrypt 匹配校验
2. newPassword != oldPassword
3. newPassword == confirmPassword
4. 更新 password_hash 和 pwd_changed_at

## 5. 业务逻辑与安全规范

### 5.1 密码安全
- 存储：BCrypt（`BCryptPasswordEncoder`，strength=12）
- 初始密码：可在系统参数（nscp_sys_param）中配置，key=`sys.default.password`
- 复杂度（前端实时校验）：≥8位，至少含大写字母、小写字母、数字、特殊字符各1个

### 5.2 账户锁定
- 连续登录失败 N 次（建议 5 次，可配置）→ 自动设置 status=LOCKED，locked_until=NOW()+锁定时长
- 登录时如果 `locked_until < NOW()`，自动解锁（status 改回 ACTIVE）
- TENANT_ADMIN 可手动解锁（PUT enable）

### 5.3 权限边界
- TENANT_ADMIN 只能管理**同 tenant_id** 的用户（MP 租户插件自动隔离）
- NORMAL 用户只能操作 `/api/sys/users/me/**`
- NORMAL 不能修改 TENANT_ADMIN 的信息

## 6. 与 scp-foundation 的主要差异

| 对比点 | scp-foundation（参考） | NEWSCP（本设计） |
|--------|----------------------|----------------|
| 密码存储 | 明文 / AES（⚠️ 安全隐患） | BCrypt（strength=12） |
| 认证机制 | Session + Cookie | JWT（无状态） |
| 用户状态 | `enabled` + `userStatus` 两字段 | `status` 单字段（ACTIVE/DISABLED/LOCKED） |
| 锁定机制 | 与外部 LoginFailCount 服务耦合 | `locked_until` 字段，登录时自判 |
| 权限声明 | 手动 if 判断 userType | `@PreAuthorize("hasAuthority(...)")` 声明式 |
| SQL 查询 | 部分使用 `${}` 拼接（⚠️ 注入风险） | 全部使用 `#{}` 参数化 |
| 多租户隔离 | 手动在 Service 层传 tenantId | MP 租户插件全自动注入 |
