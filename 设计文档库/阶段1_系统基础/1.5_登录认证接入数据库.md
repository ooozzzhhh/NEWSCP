# 1.5 登录认证接入数据库用户体系

> 版本：v1.0 | 日期：2026-04-14 | 所属阶段：阶段1 - 系统基础

## 1. 功能概述

当前 `AuthService` 使用硬编码 `admin/123456` 进行登录验证。本文档描述如何将登录认证切换为真实的数据库用户体系，同时增补密码错误锁定、登录日志、密码复杂度等生产级能力。

**前置条件：** 必须在 `nscp_sys_user` 表（见 1.1）和 `nscp_sys_role/permission`（见 1.2）建好并写入初始数据后，才能执行本改造。

**改造范围：**
- 后端：`AuthService`、`JwtTokenProvider`、`JwtAuthenticationFilter`、新增 `UserDetailsServiceImpl`
- 前端：登录页去掉预填默认凭证

---

## 2. 改造后的登录流程

```
前端 POST /auth/login { username, password, tenantId }
  │
  ▼
AuthController.login()
  │
  ├─ 1. 检查 tenantId（可选：若未传，从 username 推断或报错）
  │
  ├─ 2. 查询 nscp_sys_user WHERE tenant_id=? AND username=? AND deleted=0
  │     若不存在 → 返回「用户名或密码错误」（不泄露"用户名不存在"）
  │
  ├─ 3. 检查用户状态
  │     status=DISABLED → 返回「账号已禁用，请联系管理员」
  │     status=LOCKED 且 locked_until > NOW() → 返回「账号已锁定，请 x 分钟后重试」
  │     status=LOCKED 且 locked_until <= NOW() → 自动解锁（status=ACTIVE，reset 失败计数）
  │
  ├─ 4. BCrypt 密码验证
  │     匹配失败 → 累加失败次数，达阈值则锁定 → 返回「用户名或密码错误，还剩 N 次机会」
  │
  ├─ 5. 密码验证成功
  │     重置失败计数
  │     查询用户角色列表（JOIN nscp_sys_user_role + nscp_sys_role）
  │
  ├─ 6. 生成 JWT
  │     payload: { sub=userId, username, tenantId, roles=[...], real_name }
  │
  └─ 7. 返回 LoginResponse { token, tokenType, expiresIn, userId, username, tenantId, roles, realName }
```

## 3. 数据库改动

### 3.1 登录失败计数（存储在 nscp_sys_user 中）

在 `nscp_sys_user` ���增加两个字段（避免建独立表）：

```sql
ALTER TABLE nscp_sys_user
  ADD COLUMN login_fail_count  INT      NOT NULL DEFAULT 0 COMMENT '连续登录失败次数',
  ADD COLUMN last_login_at     DATETIME                    COMMENT '最后登录成功时间';
```

> **为什么不建独立表：** scp-foundation 有独立的 LoginFailCountService，但当前产品体量下单独建服务增加复杂度。失败次数和锁定逻辑与用户强相关，放在 nscp_sys_user 中更直接，后期如需分布式计数再迁移 Redis。

### 3.2 MybatisPlusConfig 忽略表更新

`nscp_sys_permission` 和 `nscp_sys_user_dept`（关联表）需要加入忽略列表：


## 4. 后端代码改造

### 4.1 AuthService 核心改造

**当前（硬编码）：**
```java
// AuthService.java - 当前实现（需废弃）
public LoginResponse login(LoginRequest request) {
    if (!"admin".equals(request.getUsername()) || !"123456".equals(request.getPassword())) {
        throw new RuntimeException("用户名或密码错误");
    }
    String token = jwtTokenProvider.generateToken("u-admin", "admin", "demo-tenant", List.of("ADMIN"));
    return new LoginResponse(token, ...);
}
```

**改造后：**
```java
@Service
@RequiredArgsConstructor
public class AuthService {

    private final SysUserMapper userMapper;
    private final SysRoleMapper roleMapper;
    private final JwtTokenProvider jwtTokenProvider;
    private final PasswordEncoder passwordEncoder;

    /** 登录失败锁定阈值（后续可改为从 nscp_sys_param 读取） */
    private static final int LOCK_THRESHOLD = 5;
    /** 锁定时长（分钟） */
    private static final int LOCK_MINUTES = 30;

    @Transactional
    public LoginResponse login(LoginRequest request) {
        String tenantId = StringUtils.hasText(request.getTenantId())
            ? request.getTenantId() : "demo-tenant";

        // 1. 查询用户（不能用 MP 租户插件，手动传 tenantId）
        SysUser user = userMapper.selectByTenantAndUsername(tenantId, request.getUsername());
        if (user == null) {
            throw new BusinessException("用户名或密码错误");
        }

        // 2. 检查用户状态
        checkUserStatus(user);

        // 3. 验证密码
        if (!passwordEncoder.matches(request.getPassword(), user.getPasswordHash())) {
            handleLoginFailure(user);
            int remaining = LOCK_THRESHOLD - user.getLoginFailCount();
            if (remaining <= 0) {
                throw new BusinessException("用户名或密码错误，账号已被锁定，请 " + LOCK_MINUTES + " 分钟后重试");
            }
            throw new BusinessException("用户名或密码错误，还剩 " + remaining + " 次机会");
        }

        // 4. 登录成功：重置失败计数，更新最后登录时间
        handleLoginSuccess(user);

        // 5. 查询角色
        List<String> roleCodes = roleMapper.selectRoleCodesByUserId(user.getId(), tenantId);

        // 6. 生成 JWT
        String token = jwtTokenProvider.generateToken(
            String.valueOf(user.getId()), user.getUsername(), tenantId, roleCodes
        );

        return LoginResponse.builder()
            .token(token)
            .tokenType("Bearer")
            .expiresIn(jwtTokenProvider.getExpirationSeconds())
            .userId(String.valueOf(user.getId()))
            .username(user.getUsername())
            .realName(user.getRealName())
            .tenantId(tenantId)
            .roles(roleCodes)
            .build();
    }

    private void checkUserStatus(SysUser user) {
        if ("DISABLED".equals(user.getStatus())) {
            throw new BusinessException("账号已禁用，请联系管理员");
        }
        if ("LOCKED".equals(user.getStatus())) {
            if (user.getLockedUntil() != null && user.getLockedUntil().isAfter(LocalDateTime.now())) {
                long minutes = Duration.between(LocalDateTime.now(), user.getLockedUntil()).toMinutes() + 1;
                throw new BusinessException("账号已锁定，请 " + minutes + " 分钟后重试");
            }
            // 锁定已到期，自动解锁
            userMapper.resetLock(user.getId());
            user.setStatus("ACTIVE");
            user.setLoginFailCount(0);
        }
    }

    private void handleLoginFailure(SysUser user) {
        int failCount = user.getLoginFailCount() + 1;
        if (failCount >= LOCK_THRESHOLD) {
            LocalDateTime lockUntil = LocalDateTime.now().plusMinutes(LOCK_MINUTES);
            userMapper.lockUser(user.getId(), lockUntil, failCount);
        } else {
            userMapper.incrementFailCount(user.getId(), failCount);
        }
    }

    private void handleLoginSuccess(SysUser user) {
        userMapper.recordLoginSuccess(user.getId(), LocalDateTime.now());
    }

    /** 获取当前登录用户信息（/auth/me） */
    public MeResponse getMe(String userId) {
        SysUser user = userMapper.selectById(Long.parseLong(userId));
        if (user == null) {
            throw new BusinessException("用户不存在");
        }
        List<String> permCodes = permissionMapper.selectPermCodesByUserId(Long.parseLong(userId));
        return MeResponse.builder()
            .userId(userId)
            .username(user.getUsername())
            .realName(user.getRealName())
            .tenantId(user.getTenantId())
            .permissions(permCodes)
            .build();
    }
}
```

### 4.2 JwtTokenProvider 扩展

在 JWT payload 中增加 `realName` 字段，方便前端不用额外查接口显示用户名：

```java
public String generateToken(String userId, String username, String tenantId, List<String> roles) {
    // 注意：角色列表放在 JWT 中仅用于快速标识，权限从 DB 实时加载
    return Jwts.builder()
        .subject(userId)
        .claim("username", username)
        .claim("tenantId", tenantId)
        .claim("roles", roles)
        .issuedAt(new Date())
        .expiration(new Date(System.currentTimeMillis() + expirationSeconds * 1000L))
        .signWith(key)
        .compact();
}
```

### 4.4 UserDetailsServiceImpl（新增）

```java
@Service
@RequiredArgsConstructor
public class UserDetailsServiceImpl implements UserDetailsService {

    private final SysUserMapper userMapper;
    private final SysPermissionMapper permissionMapper;

    @Override
    public UserDetails loadUserByUsername(String userId) throws UsernameNotFoundException {
        SysUser user = userMapper.selectById(Long.parseLong(userId));
        if (user == null || user.getDeleted() == 1) {
            throw new UsernameNotFoundException("用户不存在: " + userId);
        }

        // 加载用户所有权限码（用于 @PreAuthorize 验证）
        List<String> permCodes = permissionMapper.selectPermCodesByUserId(Long.parseLong(userId));
        List<GrantedAuthority> authorities = permCodes.stream()
            .map(SimpleGrantedAuthority::new)
            .collect(Collectors.toList());

        return new org.springframework.security.core.userdetails.User(
            userId,
            user.getPasswordHash(),
            "ACTIVE".equals(user.getStatus()),  // enabled
            true,                                // accountNonExpired
            true,                                // credentialsNonExpired
            !"LOCKED".equals(user.getStatus()),  // accountNonLocked
            authorities
        );
    }
}
```

```java
public ApiResponse<MeResponse> getMe() {
```

**SecurityUtils 工具类（新增）：**
```java
public final class SecurityUtils {
    public static String getCurrentUserId() {
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        if (auth == null || !auth.isAuthenticated()) {
            throw new BusinessException("未登录");
        }
        return (String) auth.getPrincipal();
    }
}
```

## 6. 前端改造

### 6.1 LoginPage.tsx 改造点

1. **去掉预填的硬编码凭证**（username/password 默认值清空）
2. 增加 tenantId 输入框的提示文案
3. 登录成功后，在存储 session 的同时调用 `/api/sys/menu` 并存储 menus/permissions

```typescript
// LoginPage.tsx 改造后的登录成功处理
async function handleSubmit(values: FormValues) {
  try {
    const res = await login(values)
    if (res.code === 0) {
      setSession(res.data)                         // 存 token/userId/etc
      const menuRes = await fetchMenu()            // 拉取菜单和权限
      if (menuRes.code === 0) {
        setMenus(menuRes.data.menus)
        setPermissions(menuRes.data.permissions)
      }
      navigate('/app/dashboard')
    }
  } catch (err) {
    setError(getErrorMessage(err))
  }
}
```

### 6.3 页面刷新时的菜单恢复

页面刷新时，Zustand + localStorage 能恢复 session，但 menus/permissions 不持久化（避免权限变更后缓存过期问题）。刷新时需重新拉取：

```typescript
// App.tsx 或路由守卫中
useEffect(() => {
  const token = useAuthStore.getState().session?.token
  if (token && menus.length === 0) {
    fetchMenu().then((res) => {
      if (res.code === 0) {
        setMenus(res.data.menus)
        setPermissions(res.data.permissions)
      }
    })
  }
}, [])
```

## 7. 改造后验证清单

开发完成后，执行以下场景验证：

| 场景 | 预期结果 |
|------|---------|
| 正确用户名+密码登录 | 返回 JWT，前端跳转 dashboard，侧边栏显示对应菜单 |
| 错误密码（1次） | 返回「用户名或密码错误，还剩 4 次机会」 |
| 错误密码（5次） | 第5次返回「账号已锁定，请 30 分钟后重试」，status=LOCKED |
| 30分钟后再次登录 | 自动解锁，正确密码可登录 |
| 禁用账号登录 | 返回「账号已禁用，请联系管理员」 |
| 无效 JWT 访问受保护接口 | 返回 401 |
| 使用有效 JWT 但无权限访问接口 | 返回 403（@PreAuthorize 拦截） |
| 菜单接口返回的菜单 | 与用户角色权限一致 |

## 8. 与 scp-foundation 的主要差异

| 对比点 | scp-foundation | NEWSCP |
|--------|---------------|--------|
| 认证方式 | Session + Cookie | JWT（无状态） |
| 密码存储 | 明文/AES | BCrypt（strength=12） |
| 失败计数 | 独立 LoginFailCountService | 字段直接存 nscp_sys_user |
| 锁定字段 | userStatus + unlockTime | status + locked_until |
| 验证码 | 支持图形验证码（可配置） | 暂不实现（后期可加） |
| 密码过期 | 支持（pwd_expire_cycle 配置） | 暂不实现（后期按需加） |
| 登录日志 | 无 | last_login_at 字段记录（后期可扩展登录日志表） |
| 注销 | session.invalidate() | JWT 无状态，前端清除 token 即可（无服务端注销）|
