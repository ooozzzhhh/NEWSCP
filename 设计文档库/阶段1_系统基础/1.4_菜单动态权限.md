# 1.4 菜单动态权限设计文档

> 版本：v1.0 | 日期：2026-04-14 | 所属阶段：阶段1 - 系统基础

## 1. 功能概述

实现：
1. **后端动态菜单下发**：登录后根据用户角色返回该用户有权访问的菜单树（取代前端静态菜单列表）
2. **前端路由动态生成**：根据菜单树动态注册路由，无权限路由访问时跳转 403
3. **按钮权限控制**：前端按钮通过权限码判断是否显示，后端接口通过 `@PreAuthorize` 保护
4. **侧边栏渲染**：侧边栏菜单与用户权限实时绑定

**依赖：** 用户管理（1.1）、角色与权限（1.2）

---

## 2. 菜单数据来源

菜单数据存储在 `nscp_sys_permission` 表中（见 1.2 角色与权限设计）。  
MENU 类型的权限节点即为菜单，BUTTON 类型为按钮权限。

**完整权限码规划（初期）：**

```
sys                              系统管理（一级菜单）
├── sys:user                     用户管理（二级菜单）
│   ├── sys:user:list            查看用户列表（按钮）
│   ├── sys:user:create          新增用户
│   ├── sys:user:update          编辑用户
│   ├── sys:user:delete          删除用户
│   └── sys:user:reset-pwd       重置密码
├── sys:role                     角色管理
│   ├── sys:role:list
│   ├── sys:role:create
│   ├── sys:role:update
│   ├── sys:role:delete
│   └── sys:role:assign-perm     分配权限
└── sys:dept                     部门管理
    ├── sys:dept:list
    ├── sys:dept:create
    ├── sys:dept:update
    └── sys:dept:delete

mds                              主数据（占位，阶段6实现）
├── mds:product                  产品管理
├── mds:stock-point              库存点管理
└── mds:customer                 客户管理

（后续阶段按需补充）
```

## 3. 后端接口设计

### 3.1 菜单接口

| 方法 | 路径 | 权限 | 功能 |
|------|------|------|------|
| GET | `/api/sys/menu` | 已登录 | 获取当前用户的菜单树 + 按钮权限码列表 |

- `menus`：只含 MENU 类型节点，构成侧边栏树
- `permissions`：所有 BUTTON 类型的权限码平铺列表，供前端做按钮级控制

### 3.3 后端实现逻辑

```java
@GetMapping("/api/sys/menu")
public ApiResponse<MenuResponse> getCurrentUserMenu() {
    // 1. 从 SecurityContext 取当前用户 ID
    String userId = SecurityUtils.getCurrentUserId();
    
    // 2. 查询用户所有有效权限（JOIN 三表：user_role + role_permission + permission）
    List<SysPermission> allPerms = permissionMapper.selectByUserId(userId);
    
    // 3. 分离 MENU 和 BUTTON
    List<SysPermission> menuPerms = allPerms.stream()
        .filter(p -> "MENU".equals(p.getPermType()))
        .collect(Collectors.toList());
    List<String> buttonCodes = allPerms.stream()
        .filter(p -> "BUTTON".equals(p.getPermType()))
        .map(SysPermission::getPermCode)
        .collect(Collectors.toList());
    
    // 4. 构建菜单树
    List<MenuVO> menuTree = buildMenuTree(menuPerms);
    
    return ApiResponse.ok(new MenuResponse(menuTree, buttonCodes));
}
```

**SQL（permissionMapper.selectByUserId）：**
```sql
SELECT DISTINCT p.*
FROM nscp_sys_user_role ur
  JOIN nscp_sys_role_permission rp ON rp.role_id = ur.role_id
  JOIN nscp_sys_permission p       ON p.id = rp.perm_id
WHERE ur.user_id = #{userId}
  AND p.deleted = 0
  AND p.status = 'ENABLED'
ORDER BY p.parent_id, p.sort_order
```

## 4. 前端实现设计

### 4.1 登录后初始化流程

```
登录成功（/auth/login 返回 token）
  → 存储 token 到 Zustand（持久化 localStorage）
  → 调用 GET /api/sys/menu
  → 将 menus 存入 Zustand（authStore.menus）
  → 将 permissions 存入 Zustand���authStore.permissions）
  → 根据 menus 动态注册路由
  → 渲染侧边栏
  → 跳转到 /app/dashboard
```

### 4.2 Zustand Store 扩展

```typescript
// stores/auth-store.ts（在现有基础上扩展）

interface MenuNode {
  id: number
  permCode: string
  permName: string
  routePath: string
  componentPath?: string
  icon?: string
  sortOrder: number
  children: MenuNode[]
}

interface AuthState {
  session: AuthSession | null
  menus: MenuNode[]           // 菜单树（用于侧边栏渲染）
  permissions: string[]       // 按钮权限码列表

  setSession: (session: AuthSession) => void
  setMenus: (menus: MenuNode[]) => void
  setPermissions: (permissions: string[]) => void
  clearSession: () => void
}
```

### 4.3 侧边栏组件改造

当前 `AppLayout.tsx` 中菜单列表是硬编码的，需改为从 Zustand store 读取：

```typescript
// layout/AppLayout.tsx
const menus = useAuthStore((s) => s.menus)

// 递归渲染菜单树
function renderMenuNode(node: MenuNode) {
  if (node.children.length > 0) {
    return (
      <MenuGroup key={node.id} label={node.permName} icon={node.icon}>
        {node.children.map(renderMenuNode)}
      </MenuGroup>
    )
  }
  return (
    <NavLink key={node.id} to={node.routePath}>
      {node.permName}
    </NavLink>
  )
}
```

### 4.4 路由动态注册

前端路由策略：**静态路由 + 动态懒加载组件**

所有可能的页面组件预先定义在路由映射表中，根据 `componentPath` 动态 import：

```typescript
// router/routeComponentMap.ts
const routeComponentMap: Record<string, React.LazyExoticComponent<any>> = {
  'system/users/index':       React.lazy(() => import('../pages/system/users/index')),
  'system/roles/index':       React.lazy(() => import('../pages/system/roles/index')),
  'system/depts/index':       React.lazy(() => import('../pages/system/depts/index')),
  'master/products/index':    React.lazy(() => import('../pages/master/products/index')),
  // 后续页面按此模式追加
}

// router/AppRouter.tsx
function buildRoutes(menus: MenuNode[]): RouteObject[] {
  const routes: RouteObject[] = []
  
  function traverse(nodes: MenuNode[]) {
    for (const node of nodes) {
      if (node.componentPath && routeComponentMap[node.componentPath]) {
        const Component = routeComponentMap[node.componentPath]
        routes.push({
          path: node.routePath.replace('/app/', ''),
          element: <Component />,
        })
      }
      if (node.children.length > 0) traverse(node.children)
    }
  }
  
  traverse(menus)
  return routes
}
```

**主路由结构：**
```typescript
<Routes>
  <Route path="/login" element={<LoginPage />} />
  <Route path="/app" element={<RequireAuth><AppLayout /></RequireAuth>}>
    <Route index element={<Navigate to="/app/dashboard" />} />
    <Route path="dashboard" element={<DashboardPage />} />
    {/* 动态路由由 buildRoutes(menus) 生成 */}
    {dynamicRoutes}
    {/* 兜底：无权限或不存在路由 */}
    <Route path="*" element={<NotFoundPage />} />
  </Route>
</Routes>
```

### 4.5 按钮权限 Hook

```typescript
// hooks/usePermission.ts
import { useAuthStore } from '../stores/auth-store'

export function useHasPermission(permCode: string): boolean {
  const permissions = useAuthStore((s) => s.permissions)
  return permissions.includes(permCode)
}

// 批量检查（AND 语义）
export function useHasAllPermissions(permCodes: string[]): boolean {
  const permissions = useAuthStore((s) => s.permissions)
  return permCodes.every((code) => permissions.includes(code))
}
```

**使用示例：**
```tsx
// pages/system/users/index.tsx
import { useHasPermission } from '../../../hooks/usePermission'

export default function UserListPage() {
  const canCreate = useHasPermission('sys:user:create')
  const canDelete = useHasPermission('sys:user:delete')

  return (
    <div>
      <div className="toolbar">
        {canCreate && (
          <Button onClick={handleCreate}>新增用户</Button>
        )}
      </div>
      <Table
        columns={[
          ...baseColumns,
          {
            key: 'action',
            render: (row) => (
              <>
                <Button onClick={() => handleEdit(row)}>编辑</Button>
                {canDelete && (
                  <Button onClick={() => handleDelete(row)}>删除</Button>
                )}
              </>
            ),
          },
        ]}
      />
    </div>
  )
}
```

### 4.6 403 页面与无权限路由处理

```typescript
// pages/ForbiddenPage.tsx
// 访问无权限路由时展示此页面

// RequireAuth 组件中增加权限路由检查
function RequirePermission({ permCode, children }: { permCode: string, children: React.ReactNode }) {
  const hasPermission = useHasPermission(permCode)
  if (!hasPermission) return <Navigate to="/app/403" replace />
  return <>{children}</>
}
```

## 5. 关键流程时序图

```
用户 → POST /auth/login
后端 → 验证密码 → 生成 JWT（含 userId/tenantId/roles）
前端 → 存 token
前端 → GET /api/sys/menu（带 token）
后端 → 从 JWT 解析 userId
后端 → 查询用户权限树（SQL JOIN 三表）
后端 → 构建 menus + permissions 返回
前端 → setMenus() + setPermissions()
前端 → 渲染侧边栏（menus）
前端 → 注册动态路由

用户 → 点击「新增用户」
前端 → useHasPermission('sys:user:create') → true → 按钮可见
前端 → POST /api/sys/users
后端 → @PreAuthorize("hasAuthority('sys:user:create')") → 通过
后端 → 执行业务逻辑
```

## 6. 菜单权限变更后的处理

| 场景 | 处理方式 |
|------|---------|
| 管理员修改了角色的权限 | 已登录用户下次请求时 @PreAuthorize 立即生效；侧边栏/按钮需刷新页面（或调用刷新菜单接口）更新 |
| 用户角色变更 | 同上；建议在权限变更后后端触发通知（WebSocket，阶段3实现），前端自动重新拉取菜单 |
| token 过期 | Axios 拦截器捕获 401 → clearSession → 跳转登录页 |

**当前（阶段1）简化方案：** 页面刷新时重新调用 `/api/sys/menu`，不做实时推送（待阶段3 WebSocket 实现后升级）。

## 7. 与 scp-foundation 的主要差异

| 对比点 | scp-foundation | NEWSCP |
|--------|---------------|--------|
| 菜单数据 | DB（auth_rbac_resource） | DB（nscp_sys_permission），无 tenant_id（系统级） |
| 租户自定义菜单 | 支持（resource.tenantId 不为 null） | 不支持（菜单树由开发人员统一维护） |
| 路由注册 | 前端静态路由 | 静态路由 + 动态懒加载（按 componentPath） |
| 模块过滤 | 按 moduleCode + moduleRelationship() 过滤 | 暂无多模块概念，单一产品全菜单 |
| 菜单国际化 | 通过 ObjectI18n 翻译 | 暂不实现 |
| 权限生效时机 | 刷新 Session | JWT 无状态，权限从 DB 实时加载（@PreAuthorize） |
