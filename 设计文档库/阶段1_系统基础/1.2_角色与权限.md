# 1.2 角色与权限设计文档

> 版本：v1.0 | 日期：2026-04-14 | 所属阶段：阶段1 - 系统基础

## 1. 功能概述

实现基于角色的访问控制（RBAC），支持：
- **角色管理**：管理员创建/编辑/删除角色，为角色分配菜单+按钮权限
- **用户-角色分配**：将角色绑定到用户
- **权限粒度**：菜单级（控制侧边栏显示）+ 按钮级（控制操作按钮显示/隐藏）
- **后端鉴权**：接口通过 `@PreAuthorize("hasAuthority('权限码')")` 声明式保护

**依赖：** 用户管理（1.1）

## 2. 数据库设计

### 2.1 nscp_sys_role（角色表）

```sql
CREATE TABLE nscp_sys_role (
    id          BIGINT          NOT NULL AUTO_INCREMENT PRIMARY KEY COMMENT '主键',
    tenant_id   VARCHAR(64)     NOT NULL                           COMMENT '租户ID',
    role_code   VARCHAR(64)     NOT NULL                           COMMENT '角色代码（租户内唯一，如 ANALYST/PLANNER）',
    role_name   VARCHAR(128)    NOT NULL                           COMMENT '角色名称',
    remark      VARCHAR(256)                                       COMMENT '备注',
    sort_order  INT             NOT NULL DEFAULT 0                 COMMENT '排序（升序）',
    created_by  VARCHAR(64)     NOT NULL                           COMMENT '创建人ID',
    created_at  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_by  VARCHAR(64)                                        COMMENT '最后修改人ID',
    updated_at  DATETIME                 ON UPDATE CURRENT_TIMESTAMP,
    deleted     TINYINT(1)      NOT NULL DEFAULT 0,

    UNIQUE KEY uk_tenant_role_code (tenant_id, role_code),
    INDEX idx_tenant (tenant_id)
) COMMENT '系统角色表';
```

### 2.2 nscp_sys_permission（权限/菜单资源表）

```sql
CREATE TABLE nscp_sys_permission (
    id              BIGINT          NOT NULL AUTO_INCREMENT PRIMARY KEY COMMENT '主键',
    parent_id       BIGINT          NOT NULL DEFAULT 0              COMMENT '父节点ID（0=根节点）',
    perm_type       VARCHAR(32)     NOT NULL                        COMMENT '类型：MENU（菜单）/ BUTTON（按钮）',
    perm_code       VARCHAR(128)    NOT NULL                        COMMENT '权限码（全局唯一，如 sys:user:list）',
    perm_name       VARCHAR(128)    NOT NULL                        COMMENT '权限名称（显示用）',
    route_path      VARCHAR(256)                                    COMMENT '前端路由路径（MENU 类型填写，如 /app/system/users）',
    component_path  VARCHAR(256)                                    COMMENT '前端组件路径（如 system/users/index）',
    icon            VARCHAR(64)                                     COMMENT '菜单图标（lucide-react 图标名）',
    sort_order      INT             NOT NULL DEFAULT 0              COMMENT '同级排序（升序）',
    is_hidden       TINYINT(1)      NOT NULL DEFAULT 0              COMMENT '是否隐藏（1=隐藏，不在侧边栏显示，但路由有效）',
    status          VARCHAR(32)     NOT NULL DEFAULT 'ENABLED'      COMMENT '状态：ENABLED/DISABLED',
    created_by      VARCHAR(64)     NOT NULL,
    created_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_by      VARCHAR(64),
    updated_at      DATETIME                 ON UPDATE CURRENT_TIMESTAMP,
    deleted         TINYINT(1)      NOT NULL DEFAULT 0,

    UNIQUE KEY uk_perm_code (perm_code),
    INDEX idx_parent (parent_id),
    INDEX idx_perm_type (perm_type)
) COMMENT '系统权限/菜单资源表（不含 tenant_id，权限树是系统级的，不允许租户自定义）';
```

> **注意：** `nscp_sys_permission` 不含 `tenant_id`，权限树由开发人员维护，是系统级配置，不属于租户数据。MyBatis-Plus 租户插件需将此表加入忽略列表。

**权限码命名规范：** `<模块>:<资源>:<动作>`

| 示例权限码 | 含义 |
|-----------|------|
| `sys:user:list` | 用户管理-列表查看 |
| `sys:user:create` | 用户管理-新增 |
| `sys:user:update` | 用户管理-编辑 |
| `sys:user:delete` | 用户管理-删除 |
| `sys:user:reset-pwd` | 用户管理-重置密码 |
| `sys:role:list` | 角色管理-列表查看 |
| `sys:dept:list` | 部门管理-列表查看 |
| `mds:product:list` | 产品主数据-列表查看 |

## 3. 后端接口设计

### 3.1 角色管理接口

| 方法 | 路径 | 所需权限 | 功能 |
|------|------|---------|------|
| GET | `/api/sys/roles` | `sys:role:list` | 分页查询角色列表（含每个角色的用户数） |
| POST | `/api/sys/roles` | `sys:role:create` | 新增角色 |
| GET | `/api/sys/roles/{id}` | `sys:role:read` | 查询角色详情（含分配的权限ID列表） |
| PUT | `/api/sys/roles/{id}` | `sys:role:update` | 修改角色基本信息 |
| DELETE | `/api/sys/roles/{id}` | `sys:role:delete` | 删除角色（有绑定用户时拒绝） |
| GET | `/api/sys/roles/all` | 已登录 | 获取全部角色列表（用于用户管理的角色下拉） |
| PUT | `/api/sys/roles/{id}/permissions` | `sys:role:assign-perm` | 为角色分配权限（全量替换） |

### 3.2 权限树接口

| 方法 | 路径 | 所需权限 | 功能 |
|------|------|---------|------|
| GET | `/api/sys/permissions/tree` | `sys:role:assign-perm` | 获取完整权限树（用于角色授权页面） |

### 3.3 关键请求/响应格式

#### 新增角色

```json
POST /api/sys/roles
{
  "roleCode": "ANALYST",
  "roleName": "供应链分析师",
  "remark": "可查看预测和订单数据",
  "sortOrder": 10,
  "permIds": [1, 2, 3, 10, 11]
}
```

**业务规则：**
- role_code 在当前租户内唯一
- 创建时可同步分配 permIds（写入 nscp_sys_role_permission）

#### 为角色分配权限（全量替换）

```json
PUT /api/sys/roles/1/permissions
{
  "permIds": [1, 2, 3, 10, 11, 20]
}
```

**业务规则：**
- 删除该角色所有旧权限关联，批量插入新关联（一个事务）
- 权限变更立即生效（JWT 中不缓存权限，每次请求从 DB 校验）

#### 权限树结构（响应）

```json
{
  "code": 0,
  "data": [
    {
      "id": 1,
      "permCode": "sys",
      "permName": "系统管理",
      "permType": "MENU",
      "sortOrder": 100,
      "children": [
        {
          "id": 2,
          "permCode": "sys:user",
          "permName": "用户管理",
          "permType": "MENU",
          "routePath": "/app/system/users",
          "sortOrder": 10,
          "children": [
            { "id": 10, "permCode": "sys:user:list",   "permName": "查看列表", "permType": "BUTTON" },
            { "id": 11, "permCode": "sys:user:create", "permName": "新增用户", "permType": "BUTTON" }
          ]
        }
      ]
    }
  ]
}
```

### 3.4 后端鉴权实现

#### Spring Security 权限加载链路

```
JWT Token → JwtAuthenticationFilter 解析 userId
→ UserDetailsServiceImpl.loadUserByUsername(userId)
→ 查询 nscp_sys_user_role + nscp_sys_role_permission
→ 将 perm_code 列表转为 GrantedAuthority 列表
→ 存入 SecurityContext
→ @PreAuthorize("hasAuthority('sys:user:create')") 生效
```

#### UserDetailsServiceImpl 核心逻辑

```java
@Service
public class UserDetailsServiceImpl implements UserDetailsService {
    
    @Override
    public UserDetails loadUserByUsername(String userId) {
        // 1. 查询用户基本信息
        SysUser user = userMapper.selectById(userId);
        if (user == null || user.getDeleted() == 1) {
            throw new UsernameNotFoundException("用户不存在");
        }
        
        // 2. 查询用户所有权限码（JOIN 三表）
        List<String> permCodes = permissionMapper.selectPermCodesByUserId(userId);
        
        // 3. 转换为 GrantedAuthority
        List<GrantedAuthority> authorities = permCodes.stream()
            .map(SimpleGrantedAuthority::new)
            .collect(Collectors.toList());
        
        return new org.springframework.security.core.userdetails.User(
            userId, user.getPasswordHash(), authorities
        );
    }
}
```

> **性能注意：** 每次请求都会加载权限，建议后期用 Spring Cache（`@Cacheable("user-permissions")`）缓存，并在角色权限变更时主动清除对应 cache。

#### Controller 鉴权示例

```java
@RestController
@RequestMapping("/api/sys/users")
public class UserController {

    @PreAuthorize("hasAuthority('sys:user:list')")
    @GetMapping
    public ApiResponse<Page<UserVO>> list(UserPageQueryDTO query) { ... }

    @PreAuthorize("hasAuthority('sys:user:create')")
    @PostMapping
    public ApiResponse<Long> create(@RequestBody @Valid UserCreateDTO dto) { ... }

    @PreAuthorize("hasAuthority('sys:user:delete')")
    @DeleteMapping("/{id}")
    public ApiResponse<Void> delete(@PathVariable Long id) { ... }
}
```


**交互：**
- 勾选父节点（MENU）自动全选子节点（BUTTON）
- 取消最后一个子节点时自动取消父节点勾选
- 点击「保存权限配置」调用 `PUT /api/sys/roles/{id}/permissions`

### 4.4 前端按钮权限控制

前端通过 Zustand store 中存储的 permissions 列表来控制按钮显示：

```typescript
// hooks/usePermission.ts
export function useHasPermission(permCode: string): boolean {
  const permissions = useAuthStore((s) => s.permissions)
  return permissions.includes(permCode)
}

// 使用示例
const canCreate = useHasPermission('sys:user:create')
return (
  <>
    {canCreate && <Button onClick={handleCreate}>新增用户</Button>}
  </>
)
```

**权限数据来源：** 登录成功后调用菜单接口（见 1.4），接口返回菜单树和 BUTTON 权限码列表，存入 Zustand。

## 5. 业务逻辑说明

### 5.1 角色删除限制
- 角色已被绑定用户时，拒绝删除，提示「该角色下存在 N 个用户，请先解除绑定再删除」
- 内置系统角色（如 TENANT_ADMIN_ROLE）禁止删除

### 5.2 权限生效时机
- 权限变更**立即对新请求生效**（每次请求从 DB 加载权限）
- 已登录用户的 JWT token 不需要重新签发，权限从 DB 实时读取

### 5.3 TENANT_ADMIN 的权限边界
- TENANT_ADMIN 可以创建/编辑/删除本租户的角色
- TENANT_ADMIN 不能为角色分配「超出自身权限范围」的权限（防止权限提升攻击）
  - 实现：分配权限前，校验目标 permIds 是否全部包含在当前操作人自身权限集中

## 6. 与 scp-foundation 的主要差异

| 对比点 | scp-foundation（参考） | NEWSCP（本设计） |
|--------|----------------------|----------------|
| 权限声明 | 手动 if 判断 userType | `@PreAuthorize("hasAuthority(...)")` |
| 权限加载 | 登录时存 Session | 每次请求从 DB 加载（无状态 JWT） |
| 权限表命名 | auth_rbac_resource | nscp_sys_permission |
| 菜单与按钮 | 同一张表，type=MENU/WIDGET | 同一张表，type=MENU/BUTTON（命名更直观） |
| 租户隔离 | permission 表有 tenant_id（允许租户自定义菜单） | permission 表无 tenant_id（系统级，不允许租户改菜单） |
| 权限复制 | 支持 copyResources 接口 | 暂不实现（可后期加） |
| 国际化菜单 | 通过 ObjectI18n 翻译菜单名 | 暂不实现（产品面向中文用户） |
